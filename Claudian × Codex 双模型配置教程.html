一、前置环境准备

在开始配置之前，请确保已完成以下准备工作：

✅ Obsidian 已安装（版本要求 v1.4.5+）
✅ Node.js 已安装（用于执行 npm 命令）
✅ ChatGPT 账号（用于 Codex 登录授权）
✅ Claude Code 的 AI 服务来源，三选一：
  • ① 官方 Claude / Anthropic 订阅
  • ② 第三方 API 中转或聚合平台的 API Key
  • ③ 零成本方案：使用免费 Ling 模型（详见本文番外章节）
✅ 可用的终端程序（Mac 使用 Terminal，Windows 使用 PowerShell）

以下 1.1 和 1.2 小节专为不熟悉 Node.js 和终端操作的用户准备，有经验的开发者可直接跳转到第二节。

1.1 Node.js 安装指南（新手专用 · 资深用户可跳过）

Node.js 是后续执行 npm install 命令的必要环境。若未安装，执行命令时会出现 "command not found: npm" 错误。

Mac 系统：在终端执行 brew install node（若未安装 Homebrew，请先访问 brew.sh 完成安装）

Windows 系统：访问 nodejs.org 下载 LTS 版本的 Windows Installer (.msi) 安装包，双击运行安装程序，安装完成后请重启计算机。

验证安装是否成功

打开终端，依次执行以下命令：

node -v npm -v

若两个命令均能正常显示版本号（例如 v20.11.0、10.2.4），则表示安装成功。

⚠️ 若出现 "command not found" 错误：99% 的情况是未重启终端，Windows 用户请确认是否已重启计算机。请重启后再次尝试。

1.2 终端使用入门（新手专用 · 资深用户可跳过）

Mac 系统：按下 ⌘ + 空格键 打开 Spotlight 搜索 → 输入 Terminal → 按回车键。显示的黑底白字窗口即为终端。

Windows 系统：按下 Win 键 → 输入 PowerShell → 按回车键。建议使用 PowerShell 而非传统的 CMD，其稳定性更佳。

💡 如何将命令粘贴到终端：Mac 使用 ⌘V，Windows PowerShell 使用 Ctrl+V，PowerShell 同样支持右键粘贴。

二、Claudian 插件安装

由于 Claudian 尚未上架 Obsidian 官方插件市场，需要通过 BRAT 插件管理工具进行安装。

本章节分为两种场景，请根据实际情况选择。

🔄 已安装 Claudian 的用户：请先更新至最新版本

⚠️ 重要提示：Codex 支持是 Claudian 在后续版本中新增的功能，旧版本无法看到 Codex 配置标签页，也找不到相关选项。若您已在较早版本安装了 Claudian，此步骤为必须操作，只有更新到最新版本才能使用 Codex 功能。

更新操作可在 BRAT 中一键完成：

打开 Obsidian 设置 → 左侧菜单滚动到底部，找到 BRAT 并点击 → 右侧滚动至「Beta plugin list」区域 → 找到 YishenTu/claudian 条目 → 点击右侧的 🔄 刷新图标（圆形箭头按钮），BRAT 将自动拉取最新版本并覆盖安装。

更新完成后请重启 Obsidian，然后直接跳转到第三节配置 Claude Code，无需阅读以下全新安装流程。

💡 如何确认版本更新是否成功：Claudian 的 Codex 功能支持始于 v2.0.x 版本。更新后可前往「第三方插件 → 已安装插件」查看 Claudian 的版本号进行确认。

🆕 首次安装用户：Claudian 完整安装流程

以下为五步安装流程，每步均配有截图说明，请按顺序操作。

1打开 Obsidian 设置界面

启动 Obsidian，查看窗口左下角，找到齿轮图标 ⚙️ 并点击。弹出的面板即为设置界面。

2关闭「安全模式」

在设置面板左侧菜单中向下滚动 → 找到「第三方插件」选项并点击进入。

页面顶部即为「安全模式」选项，默认处于开启状态。启用安全模式时，Obsidian 禁止安装任何第三方插件。请点击右侧开关将其关闭，在弹出的确认对话框中点击确认。

关闭安全模式后，下方的「社区插件市场」将变为可用状态，右侧的「浏览」按钮即为插件市场入口。

3安装 BRAT 插件

点击「浏览」按钮进入插件市场。

在搜索框中输入 brat，第一个搜索结果即为目标插件。插件名称为 BRAT，作者为 TfTHacker，下载量超过 68 万，请确认信息无误后再进行安装。

点击进入插件详情页 → 点击右侧的「安装」按钮 → 安装完成后按钮将变为「启用」，请再次点击启用插件。

4通过 BRAT 添加 Claudian 插件

返回设置主页面，左侧菜单滚动至底部，在「第三方插件」分类下将显示所有已安装的插件列表，找到 BRAT 并点击进入。

⚠️ 注意：BRAT 的设置界面为英文 UI，与 Obsidian 主设置的中文界面不同。无需担心，按照以下四步操作即可。

❶ 在左侧菜单底部找到 BRAT 并点击进入
❷ 在右侧页面滚动至「Beta plugin list」区域，点击紫色按钮「Add beta plugin」

此时将弹出 "Github repository for beta plugin" 对话框，请继续按以下步骤操作。

❸ 在 Repository 输入框中完整粘贴以下地址：

https://github.com/YishenTu/claudian

❹ 点击紫色按钮「Add plugin」→ 等待数秒。屏幕将显示绿色提示信息，表示 Claudian 已自动安装并启用。

5验证 Claudian 已成功启用

返回「第三方插件」主页面（注意：是上方的总开关页面，而非下方分类中的 BRAT 设置），滚动至「已安装插件」区域，确认已安装的 Claudian 右侧开关处于点亮状态，即表示安装并启用成功。

Claudian 官方功能说明："将 Claude Code 作为 AI 协作工具嵌入您的知识库。您的知识库将成为 Claude 的工作目录，赋予其完整的智能体能力：文件读写、搜索、终端命令执行以及多步骤工作流。"

启用成功后，请查看 Obsidian最左侧的侧边栏（图标列），应会新增一个 🤖 机器人图标，这就是 Claudian 对话面板的入口。

⚠️ 若未看到 🤖 图标：请完全退出 Obsidian（Mac 使用 ⌘Q，Windows 关闭窗口）后重新启动。

至此，Claudian 插件框架已安装完成。但此时尚无法进行对话，因为还未配置 AI 模型引擎。

以下两节将分别讲解如何配置 Claude Code 和 Codex 通道。

三、通道配置一：Claude Code 设置

Claude Code 是 Anthropic 官方推出的命令行工具，作为 Claudian 运行 Claude（Sonnet / Opus）模型的底层引擎。

Step 1：安装 Claude Code 命令行工具

打开终端（若不清楚如何打开，请参考 1.2 小节），执行以下命令：

npm install -g @anthropic-ai/claude-code

安装过程将持续一段时间，当光标重新回到 $ 提示符时即表示安装完成。

Step 2：Claude Code 登录与使用配置

根据您拥有的资源，Claude Code 的 AI 模型接入分为三种方式。请选择适合您的方案，Step 3、4、5 对所有方案均通用。

✅ 方案 A：拥有官方 Claude / Anthropic 订阅

在终端直接执行：

claude

首次启动将弹出登录引导：浏览器将自动打开 Claude 官方网站 → 使用您的账号登录 → 完成授权 → 自动跳转回终端，看到欢迎界面即表示登录成功。登录凭证将保存在本地 ~/.claude/ 目录下，后续调用可直接复用。

💡 若浏览器未自动打开：终端将输出一行授权链接，请复制到浏览器中手动打开即可。

完成后按 Ctrl+C 或输入 /exit 退出 CLI 交互界面，继续执行 Step 3。

✅ 方案 B：使用第三方 API 中转 / 聚合平台

若您已拥有第三方中转或聚合平台的 API Key，请按照对应平台的官方文档配置 ANTHROPIC_BASE_URL 和 ANTHROPIC_API_KEY 环境变量。由于各平台的配置字段略有差异，请以平台官方文档为准，本文不做详细展开。配置完成后直接进入 Step 3。

⭐ 方案 C：暂无任何订阅？请参考本文末尾的「番外」章节

若您既无官方订阅，也不打算使用中转服务，无需担心。

本文末尾的「番外：零成本方案 — 使用免费 Ling 模型运行 Claude Code」章节，将详细讲解如何通过 OpenRouter 平台提供的免费模型 Ling-2.6-1T，配合 CC Switch 工具一键切换 API 提供商，实现零成本、无需信用卡，仅需 5 分钟即可让 Claude Code 正常运行。

该国产开源模型访问稳定，足以满足日常编程辅助和写作需求。

对于预算有限、暂时未订阅服务或希望先进行试用的用户，推荐选择此方案：请直接滚动到番外章节完成配置后，再返回 Step 3 继续操作，无需执行本 Step 2 的内容。

⚠️ 若出现 "command not found: claude" 错误：说明 Step 1 的 CLI 安装未成功，或新安装的 CLI 尚未加入当前终端的 PATH 环境变量。请关闭终端窗口后重新打开再试。

Step 3：获取 CLI 可执行文件路径

在终端执行：

where claude

命令将输出一行路径：

您的输出路径可能与示例不同，这是正常现象，请完整复制这一行路径，稍后需要粘贴到 Claudian 设置中。

💡 若命令无输出：请重启终端后再次尝试；Linux 系统若 where 命令无法找到，请改用 which claude。

Step 4：配置 Claudian 设置

返回 Obsidian → 点击左下角齿轮 ⚙️ 图标 → 左侧菜单滚动到底部找到 Claudian 并点击进入。

进入 Claudian 设置界面后，建议先将语言切换为中文，然后注意顶部有三个标签页：「通用」「Claude」「Codex」。请先切换到「Claude」标签页。

在「设置」区域的第一栏「Claude CLI 路径」中，将 Step 3 中复制的路径完整粘贴进去。

页面下方还有「安全」「模型」「命令与技能」等区域，全部保持默认设置即可。以下选项仅针对特定用户需要调整：

  • 「加载用户 Claude 设置」：已在终端配置过 ~/.claude/settings.json 的用户可开启
  • 「模型」区域：Max / Team / Enterprise 套餐用户可根据需要调整 1M 上下文或自定义模型，普通用户请全部忽略

Step 5：★ 功能验证

返回 Obsidian 主界面 → 点击左侧 🤖 图标打开 Claudian 主面板 → 发送测试消息如「你好」。

若收到 AI 回复，则表示 Claude Code 通道配置成功。

番外：零成本方案 — 使用免费 Ling 模型运行 Claude Code

完成主流程配置后，Claude Code 默认将根据您的 Claude / Anthropic 订阅进行计费。若您希望先零成本试用，或预算有限、暂时未订阅服务，可选择此方案：通过 OpenRouter 平台的免费模型 Ling-2.6-1T，配合 CC Switch 工具一键切换 API 提供商。Claude Code 本身保持不变，底层使用的是免费的 Token 额度。

💡 关于 CC Switch：这是一款跨平台桌面应用，专门用于管理 Claude Code、Codex、Gemini CLI、OpenCode、OpenClaw 等 AI 编程工具的 API 提供商。其优势在于无需手动编辑配置文件，所有操作均可通过可视化界面完成。

Step 1：在 OpenRouter 获取 API Key

访问 openrouter.ai → 使用 Google 账号或邮箱注册 → 进入 openrouter.ai/keys 页面 → 点击「Create Key」。

在弹出的对话框中，仅需填写 Name 字段（可任意命名，例如 Obsidian Use Key，便于后续识别），下方的 Credit limit、Reset limit every、Expiration 等选项全部保持默认即可。点击「Create」→ 完整复制生成的 API Key。

⚠️ API Key 仅显示一次，请妥善保存。若遗失需要重新生成。

Step 2：查找免费模型 Ling-2.6-1T 并复制模型 ID

❶ 在 OpenRouter 顶部菜单进入 Models 页面
❷ 在搜索框输入 ling
❸ 第一个搜索结果即为 inclusionAI: Ling-2.6-1T (free)（标记为 Free，输入输出均不计费）
❹ 点击模型名称右侧的复制图标，复制得到的模型 ID 为：

inclusionai/ling-2.6-1t:free
💡 OpenRouter 平台提供多款免费模型，Ling-2.6-1T 是其中表现较为出色的一款。即使免费期结束，根据当前公开的定价，其使用成本也属于非常经济的档位。

Step 3：安装 CC Switch

Mac 系统：在终端依次执行以下命令：

brew tap farion1231/ccswitch brew install --cask cc-switch

Windows 系统：访问 CC Switch Releases 下载 .msi 安装包，双击运行安装。

Step 4：在 CC Switch 中添加 OpenRouter 提供商

打开 CC Switch → 顶部标签页选择「Claude 供应商」→ 点击「添加新供应商」（CC Switch 内置 50+ 提供商预设）。

❶ 在预设列表中选择 OpenRouter（位于第二行靠右位置），选择后上方的「供应商名称」和「实际地址」（https://openrouter.ai）将自动填充，无需手动修改。

❷ 在 API Key 字段 → 粘贴上一步复制的 OpenRouter Key。

❸ 将模型名称 inclusionai/ling-2.6-1t:free 填入以下 5 个字段中（每个字段均填入相同的模型名称）：

  • 主模型
  • 推理模型 (Thinking)
  • Haiku 默认模型
  • Sonnet 默认模型
  • Opus 默认模型
⚠️ 此步骤为关键配置：CC Switch 将 Claude Code 内部使用的所有模型角色都映射到底层提供商进行处理，因此 Haiku、Sonnet、Opus 三个档位必须全部指向 Ling 模型。缺少任何一个，对应档位的调用都将失败。

❹ 点击右下角的蓝色「添加」按钮。

Step 5：启用 OpenRouter 提供商

返回 CC Switch 主界面，在刚添加的 OpenRouter 提供商卡片右侧，有蓝色的「▶ 启用」按钮，请点击启用。

💡 Claude Code 切换提供商后无需重启，配置将立即生效。也可以通过系统托盘右键菜单直接进行切换。

Step 6：★ 零成本方案验证

返回 Claudian 主面板 → 选择 Claude 组的任意模型（Opus / Sonnet / Haiku 均可，实际底层都将调用 Ling 模型）→ 发送测试消息如「你好」。

若收到 AI 回复，则表示零成本通道配置成功。此时您正在使用的是 Ling-2.6-1T 模型，完全不会产生任何费用。

四、通道配置二：Codex 设置（本文核心内容）

无论您选择的是官方订阅的主线路，还是零成本的番外方案，至此 Claude Code 应已可正常使用。若您仅需使用 Claude 模型，配置工作可到此结束。

但 Claudian 还支持另一种强大的配置方式：额外添加一条 Codex 通道，实现两个模型的随时切换。这部分功能在官方 README 中几乎没有说明，网上也难以找到完整的配置教程。以下为经过实际验证可成功运行的完整流程。

Step 1：安装 Codex 命令行工具

在终端执行：

npm install -g @openai/codex
💡 若已安装 OpenAI 的 Codex 桌面应用：也可以直接向其发送指令「帮我安装 codex cli」，它将自动为您执行上述命令，适合不愿接触命令行的用户。

Step 2：通过 ChatGPT 进行登录授权

💡 这是 Codex CLI 与 Claude Code 在使用体验上最显著的区别：Codex 使用 ChatGPT 官方授权机制，无需单独申请 OpenAI API Key，只要拥有 ChatGPT 账号即可使用。

与上一节流程相同，请先完成 CLI 登录再配置 Claudian。在终端执行：

codex login

进入交互界面后，选择「Sign in with ChatGPT」。

系统将自动打开浏览器 → 使用您的 ChatGPT 账号登录 → 完成授权 → 跳转回终端，看到 "Sign in successful" 提示即表示登录成功。

Step 3：获取 Codex CLI 可执行文件路径

在终端执行：

where codex

命令将输出一行路径：

示例路径为 /Users/rongshi/.local/bin/codex，与 Claude 类似，您看到的具体路径可能不同，这是正常现象，完整复制该路径即可。

Step 4：配置 Claudian Codex 设置（关键步骤）

返回 Obsidian → 进入 Claudian 设置界面 → 顶部标签页切换到「Codex」。

此步骤必须完成以下两个操作，缺少任意一个，Codex 都不会出现在模型切换菜单中：

❶ 打开「Enable Codex provider」开关

⚠️ 这是 Codex 通道的总开关，默认处于关闭状态。只有打开此开关，Codex 模型才会出现在 Claudian 的模型选择器中。这一步是绝大多数网上教程都未提及的关键"踩坑点"。若最终切换菜单中看不到 Codex 选项，99% 的原因是此开关未打开。

❷ 在「Codex CLI path」字段，将 Step 3 中复制的 codex 路径完整粘贴进去。

Step 5：★ Codex 通道验证

返回 Claudian 主面板 → 切换到 Codex 模型 → 发送测试消息如「你好」。

若收到 AI 回复，则表示Codex 通道配置成功。

⚠️ 若切换菜单中看不到 Codex 选项：
1️⃣ 检查 Step 4 中的 Enable Codex provider 开关是否已打开
2️⃣ 完全关闭 Obsidian 后重新启动
3️⃣ 在 Claudian 中新建一个对话框即可

五、双模型切换：左下角一键选择

两条通道均配置完成后，切换入口位于 Claudian 对话框左下角的模型选择器。点击将弹出下拉菜单，其中分为两组：

  • CLAUDE 组：Opus / Sonnet / Haiku
  • CODEX 组：GPT-5.5 / GPT-5.4 Mini
⚠️ 若下拉菜单中仅显示 CLAUDE 组，看不到 CODEX 组：
1️⃣ 检查 Step 4 ❶ 中的 Enable Codex provider 开关是否已打开
2️⃣ 完全关闭 Obsidian 后重新启动
